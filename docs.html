Post-Mortems

Crash ScriptProperties — JSON.parse(null) Incident

Problème initial : Le stockage ScriptProperties était saturé (limite de 9ko atteinte), empêchant l'ajout de nouveaux emails.

Erreur humaine : Tentative de "vider" la base manuellement en supprimant la propriété ou en effaçant son contenu.

Conséquence critique : La propriété s'est retrouvée à l'état null ou chaîne vide "". Or, le code utilisait JSON.parse(). Comme JSON.parse(null) ou JSON.parse("") provoque une erreur de syntaxe immédiate, l'add-on a crashé silencieusement.

C'est comme avoir supprimé l'étagère au lieu d'enlever les livres.

Solution : Exécuter un script de maintenance pour réinitialiser explicitement la propriété avec la chaîne "[]" (un tableau JSON vide valide) pour que le code puisse à nouveau lire et écrire dedans.
