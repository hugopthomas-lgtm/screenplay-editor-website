Product Overview — Janvier 2026
Pitch
Générer des captures d'écran photoréalistes de films qui n'existent pas encore — à partir d'un scénario ou d'une simple description de scène.
Problème & Solution

Problème

Pitcher un film sans images = vendre une maison sans photos.

  • Illustrateur : cher, lent
  • Références films existants : imprécis
  • Midjourney solo : courbe d'apprentissage, résultats incohérents

Solution

Générer en quelques clics des images qui ressemblent à des captures de film — bon cadrage, bonne lumière, bon grain.

En langage naturel, sans expertise technique.

Cible
Scénaristes Réalisateurs Producteurs Showrunners Écoles de cinéma Agences pub/créa
Fonctionnalités
Killer Feature Upload de script PDF

L'utilisateur uploade son scénario. L'IA parse les didascalies et génère une image par scène clé. Le producteur voit le film en 30 secondes.

Core Paramètres cinématographiques complets

Caméra

Plans (ECU → Establishing), Angles, Focales (14mm → 200mm), Aspect ratios

Lumière

Heure (aube → nuit), Météo, Style (naturel, chiaroscuro, néon), Mood

Film Stock

Digital (ARRI, RED), Pellicule (35mm, 16mm, 8mm), Vintage (Technicolor, VHS)

Style visuel

30+ réalisateurs, 6 DPs, Genres, Époques

Core Cohérence des personnages

Upload photos de référence. LoRAs entraînés à la volée en backend. Jusqu'à 4 personnages par projet.

Pro Export Shot List

Métadonnées techniques exportables (Excel/PDF). "Scène 12 — INT. NUIT — 35mm — Plan américain". Outil de dépouillement, pas juste d'illustration.

Moat Sauce secrète défendable

Prompts propriétaires (jamais exposés). Post-traitement réel : grain film, aberrations chromatiques, halation ajoutés par-dessus l'image. Non copiable, non visible.

Stack technique

Génération

API Midjourney (proxy) ou Replicate (Flux, SDXL). Abstraction backend pour switcher.

Cohérence personnages

LoRAs via Replicate/modal.com, ou IP-Adapter/InstantID

Post-traitement

Pipeline Python (Pillow, OpenCV) pour grain, aberrations, halation

Parsing script

pdfplumber + Claude API pour identifier didascalies/personnages/décors

Risques
Dépendance APIs génération (Midjourney pas d'API officielle)
Légalité "styles réalisateurs" (noms de réalisateurs vivants)
Qualité cohérence personnages (LoRAs rapides imparfaits)
Marché de niche (mais fort pouvoir d'achat)
Verdict
Go. Marché de niche à fort pouvoir d'achat. Aucun concurrent direct. Clé = exécution UI pro + vraie sauce secrète technique.
